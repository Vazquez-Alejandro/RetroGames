Primeros pasos:

    Preparar el terreno:

    Crear los componentes "hijos":Dentro de la carpeta /src creá:

     Un archivo llamado Encabezado.jsx que represente a un título h1

⚙️Ejercicio práctico
Obligatorio
Lista de asistentes
Tu tarea será crear una pequeña aplicación que muestre una lista de asistentes a un evento. Para esto, vas a necesitar crear un componente reutilizable (Asistente) que muestre los datos de una persona y usarlo dentro de App.jsx para renderizar la lista completa a partir de un array de datos.
Pasos a seguir:
1

    Componente asistente.jsx:

    Creá un nuevo archivo /src/Asistente.jsx.

    Este componente debe recibir tres props: nombre, tarea y emoji.

    Adentro, tiene que mostrar esos datos. Por ejemplo, en un div que contenga un h3 para el nombre y un p para la tarea con su emoji. ¡Usá la desestructuración de props que vimos!

2
Componente App.jsx:

    Dentro de App.jsx, creá un array de objetos. Cada objeto va a representar un asistente y va a tener las propiedades nombre, tarea y emoji. Podés usar este de ejemplo

const asistentes = [ 
{ nombre: 'Juan Pérez', tarea: 'Frontend Developer', emoji: '👨‍💻' },
{ nombre: 'Ana Gómez', tarea: 'Diseñadora UX/UI', emoji: '🎨' },
{ nombre: 'Carlos Ruiz', tarea: 'Backend Developer', emoji: '👩‍💻' }
];

    Usá el método .map() sobre el array asistentes para generar un componente <Asistente /> por cada persona en la lista.

    No te olvides de pasarle a cada <Asistente /> las props correspondientes y la key única.

⚙️Ejercicio práctico

Requisitos del proyecto:
1
La fuente de datos - nuestro archivo JSON:

    Dentro de la carpeta correspondiente creá un archivo nosotros.json.

    nosotros(inicializado como [])

    error (inicializado como null)

⚙️Ejercicio Práctico
Obligatorio
Requisitos:
Añadir y gestionar el estado de carga en NewProductContainer.jsx
1
Crear el estado loading:
Justo debajo de tus otros estados, agregá uno nuevo para controlar el estado de carga. Inicialízalo en false.
2
Activar el estado de carga: 
Dentro de la función handleFormSubmit, la primera acción que debes realizar es cambiar el estado loading a true.
3
Desactivar el estado de carga: 
Es crucial desactivar el estado de carga (setLoading(false)) cuando el proceso finaliza, ya sea con éxito o con un error. Asegúrate de añadirlo al final del bloque try y también en el bloque catch. Usar finally es una excelente práctica para este caso, ya que el código dentro de este bloque se ejecutará siempre, sin importar si el try tuvo éxito o si se produjo un catch.
4
Pasar el estado como prop: 
Envía el estado loading al componente ProductForm a través de sus props.

Modificar el botón de envío:
